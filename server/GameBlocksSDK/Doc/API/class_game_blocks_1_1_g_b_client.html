Detailed Description

Main class used by a game server to connect to and interact with GameBlocks servers. This class is not guaranteed to be thread safe.

EGBClientError GameBlocks::GBClient::Connect ( const char *  gbServerAddress,
unsigned short  gbServerPort,
const GBPublicSourceId &  sourceId = 0,
const int  localPort = 0,
const bool  shouldRetryWithIncrPort = false 
)

Connects to the remote GameBlocks server with directly provided connection data.

Asynchronous. If using this method it is recommended that the passed in properties are made available in a configuration file to allow for easy deployed changes.

Enables any other components specified via LoadConfigFromXML.

WINSOCK must be initialized via WSAstartup before calling Connect. Connect must must be called before calling Tick() or interacting with incoming/outgoing events. If Connect succeeds but later the connection is lost, Ticking the GBClient will try to automatically reconnect via the reconnect functionality. Connect attempts to create a non blocking, lingering, streaming TCP socket connection. Call the Client's GetLastSocketError() function to check error codes.

Parameters
gbServerAddressAddress of the remote GameBlocks server.
gbServerPortGameBlocks remote server port.
localPortOutgoing TCP connection port. Defaults to 0 for first available.
shouldRetryWithIncrPortIf connectPort is greater than zero and the connection attempt fails and this parameter is set to true, then reconnect attempts will increment the given port each attempt until the connection passes.
sourceIdThe identifier for a Game Server Source. Can be set to NULL if only one Source exists per ip address which is connected to GameBlocks, or if individual unique Source IDs are supplied with each event which is sent to the GameBlocks server.
Returns
error code

EGBClientError GameBlocks::GBClient::PrepareEventForSending ( const char *  eventName,
const GBPublicSourceId &  sourceId,
const GBPublicPlayerId &  playerId 
)

Preparing an event clears out any previous key/value pairs which were added for sending.

To send an event, you must first prepare it, then push on key/value pairs as needed, then send the event. For example if a player joins, you can send an event named PlayerJoin with 1 key value pair: key: "PlayerName" value: "Player1" The Source and PlayerIds are packaged in the event. Events with no pairs are supported.

Parameters
eventNameThe main name of the event. Should match an event name supported by the GameBlocks back end
sourceIdDefines which game server this event is originating from for this. NULL will use the default (supplied with the Connect() call).
playerIdUnique Player identifier for the player target this event corresponds to. NULL can be used for events that do not map to a player. GBClient. If your title only has one game server per GBClient connection, this can be set to NULL or 0 to use the connection address information as server identity.
Returns
error code
