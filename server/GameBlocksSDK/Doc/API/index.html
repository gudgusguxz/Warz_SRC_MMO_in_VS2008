Welcome to the GameBlocks SDK! This documentation covers the C++ interface which is exposed in the client API, as well as environment and process setup.

Overview

Below are descriptions for the core pieces of functionality for GameBlocks.

GameBlocks Client
The GameBlocks Client is the portion of GameBlocks which is integrated into your Game Server. It provides a streamlined interface to connect to the GameBlocks Server, send game Events, and receive Actions.

GameBlocks Server
The GameBlocks Server is the portion of GameBlocks which manages your Game Title's database, connections to GameBlocks Clients, and it runs the various components of GameBlocks which allow you to gain visibility on your Game Servers and act upon what you see via Rules and Actions.

Events
Events are sent to and from the GameBlocks Client and the GameBlocks Server. They are named and contain an optional list of KeyValue pairs, as well as other basic functional information.

Sources & Targets
All Events originate from a Source, usually a Game Server. Some integrations of GameBlocks may have multiple Sources per GBClient. Sources contain groups of players. An Event also has a Target indicating the object which the Event is related to. In many cases this is the Player.

Reserved Events
These are core events used to provide basic GameBlocks functionality like Server Population and Status visibility, player movement and pattern recognition, and more. A streamlined interface to send these events is provided within PSReservedEvents.h. In addition there is a GBTitleInterface helper class which can be used by your integration to automate sending of timed events and the processing of incoming events.

Server Rules
Although not part of the Client integration, the Server Rules are important as they rely on the Client events having been sent properly. The Rules System uses the Event data sent from the client to make all of the other GameBlocks components possible.

Actions
The Rules system will run Actions for a Rule that has met its conditions. Some actions, such as emails and reports, are processed entirely on the GameBlocks server, and some actions, such as kick, player message, and sendscreenshot will send events to the Game Server GameBlocks Client for further processing.

Client Actions
Client Actions are directed at the Client as opposed to an Email action which does not involve the Client. Your integration must handle actions to Ban, Message, or Kick the player, in addition to any Custom Actions which your title defines. The Examples demonstrate the handling of incoming Actions.

Custom Client Actions
You may also create custom actions that are specific to your title and may call these whatever you like. For example you could create an action called "Teleport", which could be configured with a certain rule on the GameBlocks back-end. On your Game Server you need to handle incoming actions named "Teleport" and process as you wish. The Examples include handling incoming Actions.
